# Спринт 03: функции, параметры, return и декомпозиция

Период: **9–15 июля 2026**. План: **20 часов**. Минимум в тяжёлую неделю: **12 часов**.

Главная цель недели: перестать писать весь код одним большим полотном и научиться делить программу на маленькие понятные функции.

## Результат недели

К концу недели ты должен уметь:

- создавать функции через `def`;
- понимать разницу между `print` и `return`;
- передавать данные в функцию через параметры;
- получать результат функции через `return`;
- отличать локальные переменные от глобальных;
- читать код, где функция вызывает другую функцию;
- вынести части проекта в функции без переписывания всего с нуля.

## Правила недели

1. Сначала 25 минут сам.
2. Если застрял — просишь подсказку, не готовое решение.
3. Готовое решение используем только если ты явно просишь.
4. После каждой функции проверяй её маленьким примером.
5. Не оптимизируй раньше времени: сначала понятно, потом красиво.
